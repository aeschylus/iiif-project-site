International Image Interoperability Framework

The IIIF Standard estbliashes a suite of consistent APIs for delivering archival-quality images over the web. Read More...

Features

  • Standardized Output Across Repositories

    The IIIF specification describes a service whereby any digital object can be made accessible in archival-quality segments. The standard extends to search and metadata surrounding digital objects.

  • Community-driven

    The IIIF is built on top of the same technologies that already drive the world's most performant digital libraries. The strong commuity of passionate open-source developers continues to evolve the standard to meet the needs of the community.

  • Granular

    The IIIF Image API provides granular control over returned images, making it ideal for front-end applications with changing needs and demanding user-interface requirements. It can back tilesets at any zoom level, crop, rotate, resize, and downsample images as needed.

  • Control Surface for Scholarly Research

    Scholars and developers can access to the full depth of available information about digital objects and the manuscripts, maps, photographs and other physical media they can represent, through metadata, discovery, and fine-grained delivery of the objects themselves.

Full Image, No Transformation

The GET reqeust above specifies the selection region, size, rotation, quality, and format of an image in a standard interface. Transformations are performed in order from left to right, and the result is returned as a raw image file from the remote repository, in the specified format.

Parameter Value
Region Full
Size Full
Rotation none
Quality native
Format jpg
